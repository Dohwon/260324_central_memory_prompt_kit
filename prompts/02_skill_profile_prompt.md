# 02. Skill Profile Prompt

## 왜 만들었나

- `SKILL.md`는 실행용 원문이라 사람이 빠르게 훑기 어렵다.
- 이 프롬프트는 각 스킬의 역할, 장점, 협업 방식, 필수조건을 공유용 문서로 분리하기 위한 것이다.

## 언제 쓰나

- 스킬 카탈로그를 만들 때
- 팀원이나 다른 모델에게 "이 스킬이 뭐하는 건지" 설명해야 할 때
- 스킬 간 연결 관계를 운영 문서로 정리하고 싶을 때

## 입력으로 준비할 것

- 스킬 이름
- 있으면 원본 `SKILL.md`
- 스킬이 다루는 도메인
- 스킬의 강점, 약점, 선행 조건
- 협업하는 다른 스킬 또는 역할

## 복붙용 프롬프트

```md
너의 역할은 "스킬 설명서 작성자"다.

목표:
- 특정 스킬의 실행용 프롬프트를 복제하지 않고,
  사람이 공유하고 이해하기 쉬운 `skill profile` 문서를 만든다.
- 결과는 다른 모델에 그대로 적용하는 시스템 프롬프트가 아니라,
  역할 설명서이자 운영 문서여야 한다.

반드시 지킬 규칙:
- 원문 `SKILL.md`가 있더라도 그대로 복사하지 말고 역할 중심으로 재구성하라.
- 이 문서는 "언제 호출하는가", "무엇을 잘하는가", "무엇을 넘겨줘야 하는가"를 설명해야 한다.
- 스킬이 특정 모델에 종속되지 않도록 써라.
- 부족한 정보는 지어내지 말고 `Unknown`으로 남겨라.

입력 정보:
- 스킬 이름:
- 스킬 소속/도메인:
- 참고 원문:
- 기대 역할:
- 잘하는 일:
- 피해야 할 일:
- 협업 대상:
- 필수 입력:

출력 파일:
- `skill_docs/{skill-name}.md`

출력 구조:
1. 한 줄 역할
2. 언제 호출할지
3. 언제 호출하지 말지
4. 강점
5. 필수 입력 정보
6. 기대 산출물
7. 협업 방식
8. handoff 기준
9. 실패 패턴 / 오남용 패턴
10. 모델별 적용 메모
11. Unknown / 추가 확인 필요 항목

추가 요구:
- 가능하면 `source path`와 `related docs`도 적어라.
- 원문 프롬프트의 문장보다 운영 의미를 우선 요약하라.
- 도구 권한이 있으면 파일을 생성하고, 없으면 최종 Markdown 본문만 출력하라.
```
