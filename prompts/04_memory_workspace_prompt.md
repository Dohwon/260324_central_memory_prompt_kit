# 04. Memory Workspace Prompt

## 왜 만들었나

- 좋은 프롬프트가 있어도 폴더 구조가 없으면 메모리와 운영 문서가 다시 흩어진다.
- 이 프롬프트는 스킬, 메모리, 운영 관리, 템플릿을 한 번에 참고할 수 있는 작업 공간을 만드는 용도다.

## 언제 쓰나

- 새 레포 또는 새 회사 운영 세트를 시작할 때
- "어디에 뭘 둬야 하는지"부터 표준화하고 싶을 때
- 사람과 AI가 같이 쓰는 폴더 구조를 만들고 싶을 때

## 입력으로 준비할 것

- 워크스페이스 이름
- 단일 프로젝트용인지, 여러 프로젝트 공용인지
- 쓰려는 모델/툴 목록
- 중앙 메모리를 둘 위치
- 운영 문서 범위

## 복붙용 프롬프트

```md
너의 역할은 "중앙 메모리 워크스페이스 설계자"다.

목표:
- 스킬 문서, 오너 문서, 회사 운영 문서, 프로젝트 운영 문서, short/mid/long/log 메모리를
  함께 관리할 수 있는 폴더 구조와 초기 Markdown 파일 세트를 설계한다.

반드시 지킬 규칙:
- 폴더 구조는 사람이 이해하기 쉬워야 한다.
- 특정 IDE 전용 구조로 고정하지 말고 Markdown 중심으로 설계하라.
- 각 폴더가 왜 존재하는지 설명하라.
- short/mid/long/log 계층은 반드시 포함하라.

입력 정보:
- 워크스페이스 이름:
- 사용 목적:
- 단일 프로젝트 / 멀티 프로젝트:
- 사용하는 모델/툴:
- 중앙 메모리 위치:
- 운영 문서에 포함할 범위:

출력 내용:
1. 추천 폴더 트리
2. 각 폴더의 역할 설명
3. 초기 생성할 Markdown 파일 목록
4. 툴별 어댑터가 필요하면 추천 위치
5. 운영 시작 순서

필수로 포함할 후보 폴더:
- `owner/`
- `skill_docs/`
- `prompts/`
- `manager_memory/short-term`
- `manager_memory/mid-term`
- `manager_memory/long-term`
- `manager_memory/logs`
- `templates/`
- `adapters/`
- `projects/`

추가 요구:
- 파일 생성 권한이 있으면 초기 빈 템플릿 본문까지 써라.
- 권한이 없으면 "어떤 파일을 어떤 내용으로 만들어야 하는지"를 Markdown으로 출력하라.
```
