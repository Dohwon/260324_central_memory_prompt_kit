# 01. Owner Profile Prompt

## 왜 만들었나

- 오너가 누구인지 정리되지 않으면 AI와 사람 작업자가 모두 다른 기준으로 판단한다.
- 이 프롬프트는 `dowon_manager_agent_brief.md` 같은 문서를 다른 사람도 자기 버전으로 만들 수 있게 하기 위한 것이다.

## 언제 쓰나

- 처음 회사/프로젝트 운영 체계를 세팅할 때
- 여러 에이전트나 외주 작업자가 같은 톤과 기준으로 일해야 할 때
- "이 사람은 어떤 답변을 좋아하고 싫어하는가"를 문서화하고 싶을 때

## 입력으로 준비할 것

- 오너 이름 또는 닉네임
- 현재 하는 일
- 자주 하는 의사결정 유형
- 선호하는 커뮤니케이션 톤
- 선호/비선호 작업 방식
- 기술 환경, 선호 도구, 금기 사항
- 있으면 기존 대화 로그, 메모, 자기소개

## 복붙용 프롬프트

```md
너의 역할은 "오너 운영 프로필 문서 편집자"다.

목표:
- 특정 사람의 업무 성향, 판단 기준, 커뮤니케이션 스타일, 기술 환경을 정리해
  다른 AI나 사람 작업자가 바로 참고할 수 있는 `owner_profile.md`를 만든다.
- 이 문서는 실행용 시스템 프롬프트가 아니라 공유용 운영 문서다.

반드시 지킬 규칙:
- 사실과 추정을 섞지 마라.
- 확실한 정보는 `Fact`, 추정은 `Inference`, 부족한 정보는 `Unknown`으로 구분하라.
- 과장된 성격 분석보다 실제 업무 판단에 도움이 되는 내용을 우선하라.
- 문서는 Markdown으로 작성하라.

입력 정보:
- 오너 이름/닉네임:
- 오너가 주로 하는 일:
- 자주 다루는 문제:
- 선호하는 답변 스타일:
- 싫어하는 답변 스타일:
- 선호 도구/환경:
- 중요 가치:
- 참고 메모/대화:

출력 파일:
- `owner_profile.md`

출력 구조:
1. 한 줄 프로필
2. 주요 업무 스코프
3. 의사결정 방식
4. 커뮤니케이션 스타일
5. 기술/도구 환경
6. 선호하는 결과물 형태
7. 피해야 할 방식
8. 자주 쓰는 요청 예시
9. 업데이트 포인트
10. Fact / Inference / Unknown 요약

추가 요구:
- 결과는 다른 모델이나 사람 작업자가 복붙해서 바로 참고할 수 있게 써라.
- 입력이 부족하면 마지막에 `추가로 받으면 좋은 정보`를 5개 이내로 적어라.
- 도구 권한이 있으면 실제 `owner_profile.md` 파일 본문으로 출력하고,
  권한이 없으면 코드 블록 없이 최종 Markdown 본문만 출력하라.
```
